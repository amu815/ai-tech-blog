## Gitとは何か？
Gitはソフトウェア開発で使用される**バージョン管理システム**です。コードの変更履歴を記録し、複数の開発者が同時に作業できるようにします。チーム開発においては、**衝突の解消**や**作業の分離**に不可欠です。

### なぜGitが必要か？
従来の開発では、コードの更新が上書きされたり、バージョン管理が困難でした。Gitを活用すると、以下のようなメリットがあります。
- **変更履歴の追跡**
- **複数人の同時作業**
- **ミスの修正が容易**

## Gitの基本コマンド
Gitを始めるには、まず基本コマンドを理解しましょう。

### 1. リポジトリの初期化
```bash
# 空のリポジトリを作成
git init

# 既存のリポジトリをクローン
git clone https://github.com/yourname/repo.git
```

### 2. 変更の追跡
```bash
# 変更をステージングエリアに追加
git add .

# コミット（変更を保存）
git commit -m "初期コミット"
```

### 3. ブランチの操作
```bash
# 新しいブランチを作成
git branch feature-1

# ブランチを切り替え
git checkout feature-1

# ブランチをマージ
git merge main
```

## チーム開発の基本フロー
チームでGitを使う場合、以下の流れが一般的です。

### 1. フォークとクローン
リモートリポジトリを自分のアカウントにコピーし（`git fork`）、ローカルにダウンロードします（`git clone`）。

### 2. ブランチの作成
新機能やバグ修正の作業は、**`main`や`master`以外のブランチ**で行いましょう。

### 3. プルリクエスト（Pull Request）
```bash
# リモートリポジトリに変更をプッシュ
git push origin feature-1
```
GitHubやGitLabでPRを作成し、他のメンバーにレビューを依頼します。

### 4. レビューとマージ
レビューが通れば、自分のブランチを`main`にマージします。

## 衝突（Conflict）の解決方法
複数人が同じファイルを編集すると、**マージの衝突**が発生する場合があります。以下のように手動で解消します。

### 1. エディタで差分を確認
Gitが`<<<<<<<`, `=======`, `>>>>>>>`のマーカーで差分を表示します。

### 2. 保持したいコードを選択
不要なマーカーと不要なコードを削除します。

### 3. コミットしてプッシュ
```bash
# 衝突を解消後、コミット
git commit -am "Conflict resolved"
```

## チーム開発のベストプラクティス
- **小さなコミットを頻繁に**：変更が追跡しやすくなります。
- **ブランチの命名規則**：例）`feature/`, `bugfix/`, `hotfix/`
- **コミットメッセージの統一**：`fix: ...`, `feat: ...`, `docs: ...`
- **定期的なプル**：リモートの最新状態をローカルに取得します（`git pull`）。
